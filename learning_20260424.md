## Server 포함 Message 전송구조

서버에 메세지 던지고 서버로 부터 메세지 번호 부여 받음
그리고 그 번호를 전송할 다른 클라이언트에 직접 전송
받은 클라이어느는 받은 번호를 서버에 다시 전송하고
서버는 그 번호에 해당하는 메세지를 다시 전송

## React Project 배포
  - Image: 설치가 가능한 프로그램
  - Container: 설치한 프로그램

### 배포용 리액트 artifact생성
  `npm run build`

### Nginx가 포함된 Docker이미지 받기
  - Docker Desktop 에서 nginx 검색후 `pull` 버튼 실행
  - Docker Desktop Images 에서 nginx 이미지가 받은 것을 확인
    아무거나 받지 말고 제일 처음에 나오는 공식 nginx 이미지를 받을 것

### 설치 할 홈페이지 게시판 소스코드와 Nginx 경로 확인
  - 개발한 소스코드: 
    -  윈도우 path => C:\Users\kst\react2\build
    -  linux는 path => C:/Users/kst/react2/build

  - 환경설정 ( 자세히 설명 X)
    - 윈도우 path => C:\Users\kst\react2\nginx.conf
    - linux path => C:/Users/kst/react2/nginx.conf
  
### Nginx 설정 
- port: 80
- server file(index.html등 소스코드) path: /usr/share/nginx/html
- configuration file path: /etc/nginx/conf.d/default.conf

### Docker Container 생성 옵션
 - Menu location: Images => Run
 - Container name: web01
 - Port mapping: 8080
 - Volume mapping
  - C:/Users/kst/react2/build      => /usr/share/nginx/html
  - C:/Users/kst/react2/nginx.conf => /etc/nginx/conf.d/default.conf
  - 
### Via cli
    #### creation
    ```cmd
    docker run -d -p 8080:80 ^
    -v C:/Users/kst/react2/nginx.conf:/etc/nginx/conf.d/default.conf ^
    -v C:/Users/kst/react2/build:/usr/share/nginx/html ^
    --name web02 nginx
    ```

    #### deletion
    ```cmd
    docker stop web02
    docker rm web02
    ```
