  - db.js의 host를 mysql db서버의 container 이름으로 변경
  - db.js의 port도 3306으로 바꿔야함

### Docker 공유 네트워크 생성
```cmd
REM docker network create <shared_network_name>
docker network create -d catac_shared_net
```

### Docker 공유 네트워크 나열
```cmd
docker network list
```

### Docker 공유 네트워크 삭제
```cmd
REM docker network rm <shared network name>
docker network rm catac_shared_net
```

### Docker 공유 네트워크에 컨테이너를 연결
```cmd
REM docker network connect <shared network name> <contaiiner name>
docker network connect catac_shared_net mysql
docker network connect catac_shared_net node2
```

  - 위의 번잡한 과정을 거치지 않으려면 로컬 네트워크 ipconfig으로 출력되는 주소로 db host를 설정

  - 로컬 개발환경과 배포 환경에 상관없이 잘 동작되도록 `DB_HOST`, `DB_PORT`를 환경변수에서 가져오도록
  - db.js를 수정하고 Docker container 생성시 이 환경변수들을 같이 설정해줌.
  - 로컬 개발환경에서는 프로젝트 최상위 폴더에 `.env` 파일 을 생성하고 거기에 환경변수 설정 
  ```js
      host: process.env.DB_HOST,
      port: process.env.DB_PORT,
  ```
  - 로컬 서버 시작시 `--env-file=.env` 를 `package.json`파일 안에 설정 해줘야 한다.
  ```json
  "scripts": {
    "start": "nodemon --exec \"node --env-file=.env\" app.js"
  },
  ```
  - `.dockerignore` 파일에 `.env`, `Dockerfile` 를 포함시켜야 Docker이미지에 파일들이 포함되지 않는다.
